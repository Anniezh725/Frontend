Aesthetics Aspects

Colour Scheme

Yellow and blue were found to be a relatively colour-blind friendly colours. After experimentation, blue was found to not match with many colours and was decided that it wasn't adequate to be used as a background. Pale yellow colour was therefore chosen to be used for the background. The colour palette of the webpage was then chosen from Adobe Colour with adjustments. I wanted to base my colour scheme on something natural and took the inspiration of the colours of a peach. I believed anagolous colours would be aesthetically pleasing with modification to create enough contrast between the background and the body of text.

Acessibility Aspects

The webpage follows the guidelines of W3C Accessibility Initiative(WAI).The webpage has been made accessible providing operable user interface with a navigation bar at the top of the page, to enable easy access and navigation through the websites for the user.
Navigation Aids:

  • The current page is also highlighted within the navigation bar for the user to see which enables easier navigation.
  • Keyboard navigation is also available (and sufficient) for users that cannot use a mouse.
  • The current page in the navigation bar is highlighted so that the user knows the page they're on
  • An clickable image/icon acting as a home button was also implemented to provide even easier access for the user to go back to the home page (index.html).

The overall page also has relatively high contrast of 5.1 (refer to image below) between the with the background colour and the contents of the page using Adobe Contrast Checker. High contrast was achieved with the use of a dark brown colour for the text to be seen with ease in the background. The peach/pink coloured coloured navigational menu bar is also complimented with white texts which also contrasts with the darker peach/pink colour.

The webpage also used a pale yellow background which is suited for vision impared users as it is a colour-blind friendly colour as mentioned above. This was also an attempt to make the page more gentle on the eyes by reducing the glare of an otherwise white screen. The use of alt for every images also enables the user to understand the content in the cases where the images cannot be seen or when screen readers are in use.

Font styles chosen were Arial and sans-serif fonts were used to keep the texts easy to read. Font sizes have also been kept relatively large for the user to be able to see the content from a distance with ease. Spacing between the words has also been taken into consideration. Paragraphs (p) and header (h1,h2,etc) elements were also used to separate headers for subtitiles and paragraphs for body of the text, following the WAI guidelines.

A Horizontal line was also used as a page divider, to separate the contents of the page and marks the end of a page to achieve a cleaner page. This prevents confusion between the footer and web contents.
